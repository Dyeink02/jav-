# 开发文档

## 1. 目的

这份文档用于后续开发与维护，目标是：

1. 新增功能更快落地
2. 出问题时更快定位
3. 降低跨模块改动带来的连锁风险

## 2. 当前真实架构（以代码为准）

### 2.1 主链路

- 主程序：`wails-shell/`
- 主业务：`wails-shell/internal/*`
- 前端：`desktop/renderer/*`

结论：当前是 `Wails + Go` 主运行形态，不是 Electron 主程序。

### 2.2 兼容链路

- `desktop/sidecar/*`：Node sidecar 兼容层
- `desktop/mainServices/*`：历史兼容资产

结论：仅用于 Cloudflare / 年龄检测等高风险兼容能力，普通业务不要继续往这里加逻辑。

## 3. 三大模块边界

### 3.1 JAV 爬虫

- 入口：`crawltask`、`crawlrunner`、`crawlrequest`、`crawloutput`、`crawlquality`
- 产物：`filmData.json`、`magnet-links.txt`、运行日志与摘要

### 3.2 视频整理

- 入口：`organizer` 模块与 renderer 对应控制器
- 依赖：优先消费稳定产物（例如番号集合/爬虫输出），不耦合爬虫运行态

### 3.3 AV 订阅

- 入口：`avsubscriptionv2` + `bridge/api_subscription_v2_*`
- 原则：业务流程独立，抓取能力桥接 JAV 爬虫能力，不自建第二套 Cloudflare 逻辑

## 4. AV 订阅当前约束（关键）

1. 更新检测：先做“仅番号扫描”并与基线对比，得到待更新番号
2. 更新抓取：桥接主爬虫能力，按页抓取后再按待更新番号回收
3. 日志：检测、抓取、回收都要有可复盘日志
4. 非必要不改 Cloudflare / 年龄检测核心链路

## 5. 开发流程（必须执行）

1. 先备份：改动前创建 `backups/<timestamp>-<topic>/`
2. 小步改动：优先低风险收口，不做大爆改
3. 本地自检：
   - `npm run build`
   - `npm run build:desktop-frontend`
   - `npm run sync:wails-frontend`
   - `go test ./...`（`wails-shell` 目录）
4. 打包验证：
   - `npm run phase1:build:exe`

## 6. 常见排障入口

### 6.1 订阅检测数量异常

先看：

- `wails-shell/internal/avsubscriptionv2/scan.go`
- `wails-shell/internal/bridge/api_subscription_v2_refresh.go`
- 订阅实时日志中的“第 N 页解析番号”

### 6.2 抓取成功但回收为空

先看：

- `wails-shell/internal/bridge/api_subscription_v2_finalize.go`
- `currentTaskOutputDir` 与实际 `run-*/` 目录是否一致

### 6.3 Cloudflare / 年龄检测失败

先看：

- `desktop/sidecar/services/crawlService.js`
- `desktop/sidecar/commandRouter.js`
- `wails-shell/internal/crawlrequest/*`

## 7. 代码维护约定

1. 新逻辑优先落在 Go 主链路
2. 渲染层只做 UI 与状态投影，不做主业务决策
3. 同一状态避免多处维护，统一单一事实来源
4. 改动时补必要结构注释（入口职责、边界、不要做什么）
